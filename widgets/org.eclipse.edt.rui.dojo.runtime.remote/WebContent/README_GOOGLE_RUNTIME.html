The application cannot load the Google runtime environment for Dojo. Make sure you are connected to the internet and that you have specified a Google API key in the WebContent/includeDojo.html file located in the com.ibm.egl.rui.dojo.runtime.google_1.7.1 project. If you do not have a Google API key, sign up for one (http://code.google.com/apis/loader/signup.html). If the problem persists, reload the page or specify another Dojo runtime environment.
The Dojo toolkit can be accessed by using a remote provider, such as Google or AOL, or imported into your workspace to run locally.
Each Dojo runtime provider project requires the Dojo widgets project to be in the EGL Build Path.

For general information about using Dojo and EGL, see Preparing to use the EGL Dojo widgets. That topic is both in the product help contents and at the product information center, which is here: http://publib.boulder.ibm.com/infocenter/rbdhelp/v8r0m0.

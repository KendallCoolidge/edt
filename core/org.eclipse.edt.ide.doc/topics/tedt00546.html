Updating code while the EGL debugger runs

The EGL debugger supports hotswapping, which means that the debugger processes the code changes that you make as you step through an application. Hotswapping lets you to test and modify your application at the same time.

Note: Hot swapping is not supported on the Rich UI platform. Similarly, the EGL debugger does not respond to changes made to variable values in the Variables view.
To take advantage of this feature, set the Enable hotswapping EGL debug preference as follows:
  1. From the main menu, click Window > Preferences. The Preferences page is displayed.
  2. Expand EGL and click Debug. The Debug dialog box is displayed.
  3. Click Enable hotswapping.
  4. Click OK to save the changes and exit the page.
If hotswapping is enabled and you change a function that is on the function stack, the debugging session continues at the first line of the changed function.

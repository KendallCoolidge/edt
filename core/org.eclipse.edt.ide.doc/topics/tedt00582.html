Starting the EGL debugger

To start the debugging process you need a launch configuration, which specifies where your logic resides and aspects of the runtime environment. For simple cases, you can use a launch configuration that is provided automatically.

Starting the debugger with a provided launch configuration

To launch a program with a provided launch configuration:
  1. In the Project Explorer view, right-click the EGL source file that includes the program to be launched. A menu is displayed.
  2. Click Debug As > EGL Java Main Application. If you are asked whether to go into the Debug perspective, click Yes. The program starts in the EGL debugger.

Creating a launch configuration and starting the debugger with it

To create a launch configuration:
  1. Click Run > Debug Configurations.The Debug Configurations dialog is displayed.
  2. At the left, right click EGL Java Main Application and click New. You can now create a configuration by using multiple tabs. The main tab is displayed.
  3. In the Name field, type a name for the launch configuration or accept the default. The content of the other fields are likely to be acceptable, too.
  4. If you are debugging a program that takes command line arguments, click the Arguments tab. List the arguments for the program, in the order that you would use on a command line.
  5. If you need to use a different JRE than the default for this project, click the JRE tab and complete the information there.
  6. If you need additional Java classes to run the program, specify the paths for those classes on the Classpath tab.
  7. If you need to add any other source files, specify them on the Source tab.
  8. To set environment variables for the duration of the debug session, type the variable names and values on the Environment tab.
  9. To define general information, for example about input sources, update the Common tab.
  10. To save your changes, click Apply. To undo your changes, click Revert.

To start the debugger with the new launch configuration, click Debug.

Accessing a launch configuration and starting the debugger with it

To access an existing launch configuration:
  1. Click the arrow next to the Debug button on the toolbar. A popup menu is displayed.
  2. Click Debug Configurations. The Debug Configurations dialog is displayed.
  3. At the left, expand EGL Java Main Applications and click the launch configuration of interest. If the launch configuration was created for you, the name is the fully qualified type; for example, myPkg.MyProgram.

    After you select a configuration, the Debug Configurations dialog is displayed.

You can update the configuration, as described in the earlier section on creating one.

To start the debugger with the currently selected configuration, click Debug.
